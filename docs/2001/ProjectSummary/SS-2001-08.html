Project Title

 Development of Grid Tools for electronic structure calculations

Project Details

The simulation of the electronic structure of materials from first principles is a well-established part of high performance computational science. The last few years have seen rapid advances in affordable desktop computing power, and it is now common for pump-priming or high-throughput simulations to be performed on these desktops. As the variety of available platforms increases, so does the time spent training people to deal with, and dealing with, the particular details of the implementation of the code on different platforms. In many cases, the end-user could be protected from issues concerning file types, job submission rules and file transfers. These "under the bonnet" issues are less important for the user whose primary focus is doing the simulations. The advent of Internet / Grid computing is one possible solution to this problem, and this proposal is for the development of a portable set of tools for the management and deployment of electronic structure calculations. In the context of EPCC's project to design a "Universal Portal Toolkit", this will equate to the development of an electronic structure UI, capable of distributing simulations across a variety of machines (SMP, Beowulf, PC MPP etc) using different codes.

Aims & Objectives
  1. To design and implement tools for submission and reporting of first-principles (fp) calculations, as aprt of EPCC's e-Portal toolkit.
  2. To become familiar with the issues involved with and mechanics of running these calculations on lomond.
  3. 3xtensions of the basic tools to cover
    • sanity checking
    • different calculation types (eg static and MD)
    • status reporting
    • basic visualisation
    • adaptation to other codes (eg CASTEP)

    Work Plan

    • Focus on a set of generic "job types" eg classical MD, ab initio MD, electronic structure of molecules, to span a range of widely-used codes.
    • As well as the issues of handling large datasets (typically 100+MB, can be as large as GB's) and file types, submission, clean-up etc there are partciular issues that could be tackled:
    • Visualisation tools : typically, the output files from a job may be used as the input files to a subsequent job (eg a coordinate file in an MD simulation). Tools to allow the user to visualise the data before re-submission would be extremely useful (eg AVS network)
    • This also implies the need for a filestore as part of the toolkit, to minimise large down- and up-loads of data.

    Possible timeline:

    • Weeks 1-2 Courses (MPI / OpenMP/ Java? / Visualisation)
    • Weeks 3-4 Familiarisation with calculations using VASP and EPCC e-Portal
    • Weeks 5-6 Design and implement basic interface that can identify an input file bundle, override certain parameters of these files (to allow continuation job / different type of calc) and specify output location.
    • Weeks 7-9 Extension of basic interface to cover some of ideas in (3) above.
    • Week 10 - report
