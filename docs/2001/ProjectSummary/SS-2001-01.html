 Enhancements to the EPCC Staff Skills Database

Project Details

The EPCC Staff Skills database is implemented using Access. Each new member of staff is asked to fill in an on-line HTML form listing all the skills stored in the database and grade them in a 1-5 scale. The data from this form are mailed to the training officer who updates the database. The training officer is also responsible for running a script which generates the HTML pages available for viewing.

The model described above has the following shortcomings:

  • The form has no direct connection with the database. This has two drawbacks:
  • The form cannot be reused by the same person, since they are not able to view their previous input;
  • Adding a skill to the database does not automatically update the form.
  • The script post-processing the database is not driven by the database. This has the following drawbacks:
  • Adding a skill to the database requires that the script be modified. In fact the script is currently inoperative because of this drawback;
  • The fact that the training officer must remember to run the script is not ideal.
The project aims to produce the following:
  • A mechanism which will allow the users to see their previous entries and update rather than re-enter their skills data. A way to highlight skills added to the database after the user last updated their skills is also desirable.
  • A mechanism by which adding a skill or a person to the database does not require reprogramming the script or the data-input form.
It should be noted that there is a technical hurdle to be overcome with joining the database with the HTML form and skills pages. The issue is that the EPCC web server is hosted on a Unix server, whereas the database is a PC application. There are various solutions to this problem; migrating the web server to a PC host should not be considered.

Work Plan

This largely depends on the route for solving the problem. The most basic deliverable is the design of a system which can overcome the aforementioned problems. In the indicative table below it is assumed that the technical person attempts to solve the problem using a Java wrapper around MySQL or a similar DBMS, in which case most of the work is on the database, but integration is guaranteed. Of course, populating the database is not tackled by this project.

Computing Skills Required

Database: The technical person will need to be competent in Access so as to undertake any necessary modifications to the database. The database can be rewritten, however, in a different DBMS if this proves beneficiary for the integration of the database with the form and the skills HTML pages. Java would be a prime candidate to consider for a wrapper around the database.

Script: The current scripts are written in VBA. Again an integrated Java application could be considered.

Form: This is written in HTML, and a CGI script mails the details to the training officer. This format could be retained, provided that the CGI script is kept synchronised with the database.
